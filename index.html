Serv-lang — Compiler & Syntax

A domain-specific language that compiles to native binaries via Go codegen. Infrastructure becomes syntax.

  • Infrastructure Primitives: route, broker, store, cache, ai — declare what you need.
  • Language Ergonomics (v3.3.0+): Built-in namespaces for exec, csv, xml, yaml, file, path, regex, math, encoding, hash, uuid, rand, url, jwt, compress, semver, duration, format, ip, dns, multipart, diff, and proto.
  • Go Interop Escape Hatches: Inline Go embeddings (@inline go) and extern binding blocks with receiver methods and package aliasing.
  • `serv dev` — One Command: Start the full local stack (Store, Queue, Cache, Gate) + hot-reload in a single command.
  • Type System: Structs, interfaces, enums, generics with constraints, null safety (T?), and union types.
  • AI-Native: ai.complete(), ai.chat(), ai.embed() + agent declarations with MCP tool bindings.
  • Concurrency: Fire-and-forget spawn with worker pool limits. async/await with parallel fan-out.

ServGate — Programmable API Gateway

WebAssembly-powered reverse proxy that inspects, redacts, and protects at the network edge. Now 100% complete core.

  • WASM Hot-Swapping & A/B Splits: Load filters dynamically without restarts. Run weighted traffic splits between WASM versions.
  • GraphQL Federation Proxy: Route queries to multiple backends, delegate selection sets, and merge JSON responses.
  • MCP Native Gateway: Inspect JSON-RPC tool calls, track token counts, and apply agent-specific rate limiting.
  • Cost-Aware LLM Routing: Dynamic fallback routing from cheaper models (GPT-4o-mini) to premium models (GPT-4).
  • Developer Portal (Playground): Expose interactive Swagger UI explorers and OpenAPI specs natively at /api/docs.
  • AI Prompt Guard & PII Redactor: Safety vector checks for injections and automatic credit card/SSN masking.

ServMesh — Library Service Discovery

Lightweight library-level service discovery, client-side load balancing, retries, and circuit breaking.

  • Registry Control Plane: Automatic instance registration, heartbeat status, and TTL eviction.
  • Client Resolver: Local endpoint caching, round-robin load balancing, and active backoff retries.
  • Circuit Breaker: Sandboxed breaker tracking failure rates with state switching (Closed/Open/Half-Open).
  • Context propagation: Built-in telemetry tracking trace paths and latency.

ServQueue — WASM-enabled Broker

Distributed STOMP-compliant message broker with server-side WASM transforms and guaranteed delivery.

  • Consumer Groups: Partition-assigned load balancing across subscribers.
  • Delayed Messages: Timer-wheel delivery via STOMP headers.
  • Compute-in-Queue: Inline WASM transform pipelines on topic events.
  • Dead Letter Queues: Auto-capture failed events with metadata diagnostics.
  • Deduplication: Sliding-window dedup by message hash.
  • Raft Clustering: HA consensus with automatic partition management.

ServStore — AI-Native S3 Storage

Distributed S3-compatible object storage with consistent hash rings and semantic indexing.

  • Semantic Search: Auto-indexed text via TF-IDF for cosine similarity queries.
  • Time Travel: Query objects at any historical timestamp via versioning.
  • Erasure Coding: Reed-Solomon data/parity for fault tolerance.
  • Compute-near-Data: Execute WASM transformations server-side.

ServCache — Distributed Cache Engine

High-performance caching service with pluggable backends (in-memory & Redis), native TLS client support, and OTel metrics.

  • Pluggable Backends: Support for thread-safe in-memory cache and external Redis/Valkey out-of-the-box.
  • Redis/Memcached TLS: Support native TLS encryption for all Redis and Memcached client connections.
  • OTel Cache Metrics: Export cache hit/miss ratio, memory fragmentation, and key eviction counts to OTel collectors and `/metrics`.
  • JSON/REST API: Simple endpoints for key-value GET, SET, DELETE, and CLEAR operations.
  • Active TTL Eviction: Safe background pruning of expired keys to guarantee memory efficiency.

ServConsole — Observability Dashboard

Unified glassmorphic control plane for monitoring, configuring, and tracing the Servverse ecosystem.

  • Custom Dashboard Builder: Drag-and-drop metric widgets (numbers, charts, tables, status). Save and share per team.
  • AI Agent Observatory: Real-time MCP tool call metrics, token cost calculators, and safety/PII guard alerts.
  • Diagnostics Terminal: Interactive in-browser CLI for direct service command execution.
  • Runbook Automation: Auto-execute remediation scripts from incident alert timelines.
  • SQL Workbench & Schema Inspector: Run queries and visualize DB schemas across SQLite, PostgreSQL, MySQL, Oracle.
  • SLO Tracker & Cost Optimizer: Error budget burn rates, compliance percentages, and infrastructure cost projections.

ServRegistry — Package Hub

Lightweight S3-backed community package registry for sharing modules.

  • ServStore Backend: Package tarballs stored in dedicated S3 bucket.
  • Zero-Config CLI: serv publish and serv install — done.
  • Version Management: Semver resolution with lockfile support.
  • Premium Dashboard: Browse modules and view guides in glassmorphic UI.

ServTunnel — Secure Dev Tunneling

Secure local tunnel relay and client for exposing local services to the internet natively.

  • Subdomain Routing: Dynamically routes traffic based on the public Host header.
  • Multiplexed WebSockets: Stream concurrent requests and responses over a single tunnel connection.
  • OTel Tracing: Integrates automatically with the shared OpenTelemetry collector.
  • Request Inspector: View and replay requests from the embedded inspector API.

ServCloud — Managed PaaS Platform

Lightweight process orchestrator and application deployment backend for the Serv ecosystem.

  • PaaS Deployment API: Deploy, build, and run code packages dynamically via HTTP POST.
  • Dynamic Gateway Routing: Registers newly deployed services with ServGate routes instantly.
  • Console Log Capture: Redirects standard output/error pipes to a memory-backed ring buffer.
  • Process Isolation: Automatically allocates free ports and isolates process limits.

ServCron — Distributed Scheduler

Dedicated scheduling coordinator for executing recurrent cron and interval tasks with syslog logging and API RBAC.

  • Lease-based Leader Election: Uses Redis keys to assign scheduled executions to a single instance.
  • API RBAC Enforcement: Enforces Role-Based Access Control on job builder and trigger APIs (requires admin privilege).
  • Execution Syslog Integration: Directs cron job stdout, exit statuses, and durations to syslog or central log drains.
  • JSON/REST API: Dynamically add, list, delete, or trigger jobs on demand.
  • OTel Context Propagation: Automatically link scheduled triggers to target request trace flows.
  • Standalone Fallback: Automatically runs in single-instance leader mode if Redis is absent.

ServTrace — Distributed Tracing Backend

Lightweight OTLP/HTTP collector with hierarchical trace tree reconstruction and waterfall visualization.

  • OTLP Ingestion: Standard /v1/traces POST endpoint. Drop-in replacement for heavy Jaeger/Tempo collectors.
  • Trace Tree Reconstruction: Automatically builds parent→child span hierarchies with duration and offset calculations.
  • REST Query APIs: List traces, get trace detail tree, clear traces. Zero-config integration with ServConsole waterfall UI.
  • Single Binary: No JVM, no Elasticsearch, no Cassandra. One Go binary with an in-memory circular buffer.

ServAuth — Identity & Access Provider

Secure identity and authentication provider supporting OIDC token issuance, account lockout, and structured audit logs.

  • OIDC/OAuth2 Provider: Standard OAuth2 token endpoint (POST /oauth/token) supporting the client_credentials grant flow.
  • Password Lockout Security: Automated account suspension for 5 minutes after 3 consecutive invalid credentials.
  • Auth Audit Logs: Generates structured JSON audit trails for all authentication events, login failures, and MFA setups.
  • Persistent Token Storage: Stores refresh tokens in a database to enable remote token revocation and active session audits.
  • Self-Service Password Reset: Secure token recovery generation and confirmation validator flows.
  • Serv-lang Builtin Integration: Compiled compiler support for auth.register(), auth.login(), and auth.currentUser().

ServDB — Database Proxy & Connection Pooler

Intelligent database proxy featuring read/write splitting, pool metrics, and mTLS verification.

  • Connection pooling: High-efficiency shared upstream connections pooler to minimize latency.
  • Pool Connection Stats: Exports connection pool usage, active/idle count, and wait duration metrics to `/metrics`.
  • mTLS Connection Checks: Supports mutual TLS verification for all database client connections.
  • Read/Write Splitting: Automatically routes read-only SELECT queries to Replica nodes and writes to Primary nodes.
  • Multi-Dialect Validation: Ensures parameter placement correctness (e.g. Postgres $1 vs. MySQL ? placeholder validation).
  • Serv-lang Integration: Direct compiler routing for database "servdb://pool-name/dbname" connection strings.

ServMail — Transactional Notification Hub

Transactional message delivery hub with disk queue persistence, DKIM signatures, and multiple output channels.

  • Disk Queue Persistence: Saves outgoing mail queues to disk to prevent email loss during restarts or crashes.
  • DKIM Outbound Signatures: Built-in support for automated DKIM signature headers and SPF verification checks.
  • Multi-Channel Delivery: Single endpoint to dispatch to SMTP email, Slack incoming webhooks, and Twilio SMS.
  • Templating Engine: Compiles and renders Go-style text templates using dynamic context payloads.
  • Serv-lang Builtin Integration: Builtin compiler support for mail.send(to, template, data) and notify(channel, msg).

ServFlow — Workflow & Saga Orchestrator

Stateful execution engine for multi-step DAG workflows, saga database persistence, and Saga rollbacks.

  • DAG Workflow Definition: Validates and executes complex multi-step dependency graphs topologically.
  • Saga State DB Storage: Persists saga states, transaction steps, and rollback progress in a distributed database instead of memory.
  • Saga Compensations: Runs compensation tasks in reverse order of completed steps upon task failure.
  • Resilient Limits: Configurable step-level execution timeouts and backoff retry policies.

ServLock — Distributed Locking & Lease Manager

Consensus-driven distributed lock coordinator supporting deadlock cycle detection, reentrancy, fencing tokens, and the servlockctl CLI tool.

  • Monorepo Integration: Includes servlockctl CLI under packages/servlockctl for acquiring, renewing, and inspecting locks from terminal scripts.
  • Multi-Backend Storage: Support distributed lock state storage in Redis, etcd, or local memory.
  • Deadlock Cycle Detection: Dynamic cycle graph inspection to identify and preempt wait deadlocks.
  • mTLS Transport Hardening: Secure transport channels natively out of the box using client/server certificates.
  • Raft-based Clustering: Distributed coordination powered by a built-in local consensus group.
use lock

lock "payment_sync" {
    log.info("Acquired reentrant distributed lock via servlockctl")
    syncPayments()
}

ServSecret — Encrypted Secrets Manager

Zero-knowledge secure secret store with automatic key rotation, JIT credentials, Shamir unsealing, and the servsecretctl CLI tool.

  • Monorepo Integration: Includes servsecretctl CLI under packages/servsecretctl for key rotation, audit logging, and secret injection into sub-processes.
  • IP CIDR & Geofencing: Fine-grained retrieval access controls based on network zone bounds.
  • Dynamic SQL/NoSQL Logins: Generate short-lived database logins dynamically and rotate them on schedule.
  • Master Key Sharding: Shamir's Secret Sharing to require multi-operator keys to unseal secret storage.
  • CLI Environment Injectors: Inject secrets directly into child process contexts: servsecretctl run --cmd "app".
use secret

let apiKey = secret.get("stripe.api_key")
let conn = secret.dbLogin("postgres-prod")

Live Showcase: Task API

A complete REST API in ~100 lines. The same in Go would be 400+.

showcase/task-api/main.srv 
server "servgate://localhost:8080?port=3000"
database "sqlite://tasks.db"
cache "in-memory"

migration "create_tasks" {
    db.query("CREATE TABLE IF NOT EXISTS tasks (...)")
}

route "GET" "/api/tasks" (req) {
    let cached = cache.get("tasks_list")
    if cached != nil { return cached }
    let rows = db.query("SELECT * FROM tasks")
    cache.set("tasks_list", rows, "30s")
    return { "tasks": rows }
}

route "POST" "/api/tasks" (req) {
    let title = json.parse(req.body).title
    db.query("INSERT INTO tasks (title) VALUES (?)", title)
    return { "created": true }
}

// Auto-cleanup every hour
every 1h {
    db.query("DELETE FROM tasks WHERE status='done' AND completed_at < datetime('now','-7 days')")
}

What this demonstrates

🛣️HTTP CRUD routes with path params
🗄️SQLite with auto-migrations
⚡In-memory cache with TTL
⏰Scheduled background cleanup
🔗Auto-registers routes with ServGate
📊OTel traces emitted automatically

Quickstart

Launch the entire platform locally in minutes.

One-Line Install (All Binaries)

# Windows (PowerShell)

$ irm https://raw.githubusercontent.com/vyuvaraj/servverse/main/scripts/install.ps1 | iex

 

# macOS / Linux

$ curl -fsSL https://raw.githubusercontent.com/vyuvaraj/servverse/main/scripts/install.sh | sh

$ servverse up

✓ All 16 services running

 

$ servverse doctor

✓ Verification passed: 17/17 binaries active

Just the Language (Compiler Only)

# Windows (Scoop)

$ scoop bucket add serv https://github.com/vyuvaraj/scoop-serv

$ scoop install serv

 

# macOS / Linux (Homebrew)

$ brew tap vyuvaraj/serv && brew install serv

 

$ serv init myapp && serv run myapp/main.srv
